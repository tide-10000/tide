# 不会编程？这里是你参与开源的第一步

> **GitHub 非编码者完全指南**  
> 作者：Tide 🌊 · 版本：v1.0 · 日期：2026-05-04  
> 语言：中文正文 + 英文术语标注  
> 本指南是**通用的**——任何开源项目都可以链接到这里，帮助非编码者参与协作。

---

## 你不需要会编程就能参与开源

如果你：

- 发现了一个 bug 但不知道怎么报告
- 有一个好想法但不知道怎么告诉开发者
- 有独特的经验（花店老板、老渔民、维修师傅）但不知道怎么分享
- 想加入一个项目但看到 GitHub 就害怕

**这份指南就是为你写的。** 只需要 15 分钟，你就能在 GitHub 上完成你的第一次参与。

---

## 一、GitHub 是什么（用你能理解的话）

```
┌─────────────────────────────────────────────┐
│  GitHub 就像一个全球共建的图书馆              │
│                                             │
│  · 每个项目 = 一本书（Repository / 仓库）     │
│  · 提建议 = 在借阅卡上写留言（Issue）          │
│  · 改内容 = 提交修改建议（Pull Request / PR）  │
│  · 讨论 = 在书上贴便签（Comment）             │
│  · 复制整本书 = Fork（分叉）                  │
└─────────────────────────────────────────────┘
```

你不需要理解所有术语。你只需要知道：**GitHub 是全世界开发者一起工作的地方，而且它也欢迎你。**

## 二、创建你的 GitHub 账号（5 分钟）

### 第 1 步：打开网站

在浏览器地址栏输入：`https://github.com`

你会看到一个页面，中间有一个注册框。

### 第 2 步：填写注册信息

| 字段 | 填什么 | 说明 |
|:---|:---|:---|
| **Username**（用户名） | 你想用的英文名，如 `flower_shop_lao_wang` | 这个会出现在你的所有留言上，建议用拼音或英文 |
| **Email**（邮箱） | 你的常用邮箱 | 用于接收通知和找回密码 |
| **Password**（密码） | 至少 8 位，含字母和数字 | 像其他网站一样设置 |

然后点击绿色的 **「Sign up for GitHub」** 按钮。

### 第 3 步：验证你不是机器人

GitHub 会出一个简单的拼图验证。拖动滑块完成即可。

### 第 4 步：验证邮箱

GitHub 会往你的邮箱发一封验证邮件。打开邮箱，点击邮件里的 **「Verify email address」** 按钮。

✅ 完成！你已经有了一个 GitHub 账号。

## 三、找到一个项目（3 分钟）

### 方法 1：直接打开项目地址

如果你已经知道项目地址（比如别人发给了你一个链接），直接在浏览器地址栏输入：

```
https://github.com/weihai-limh/text-cli
```

### 方法 2：搜索项目

1. 登录 GitHub 后，页面顶部有一个搜索框
2. 输入项目名称，比如 `text-cli`
3. 在搜索结果中点击正确的项目

### 到达项目页面后

你会看到：

```
┌─────────────────────────────────────────────┐
│  weihai-limh / text-cli         ⭐ 123      │  ← 项目名 + 星标数
│                                             │
│  文本驱动的 AI 技能市场...                    │  ← 项目简介
│                                             │
│  [📁 docs/] [📁 server/] [📄 README.md]     │  ← 文件列表
│                                             │
│  ┌──────────┐  ┌──────────┐                │
│  │  Issues  │  │  Code    │  ...           │  ← 顶部标签栏
│  └──────────┘  └──────────┘                │
└─────────────────────────────────────────────┘
```

**点击顶部的「Issues」标签**——这就是你参与项目的入口。

## 四、创建你的第一条 Issue（10 分钟）

Issue（议题）就像在项目的留言簿上写一条消息。它可以是 bug 报告、功能建议、经验分享、或者任何你想说的话。

### 第 1 步：进入 Issues 页面

点击项目顶部的 **「Issues」** 标签。

### 第 2 步：点击绿色按钮

你会看到一个绿色的 **「New Issue」** 按钮，点击它。

### 第 3 步：选择模板（如果有）

有些项目会提供 Issue 模板（Issue Template），让非编码者更容易参与。

如果项目有模板，你会看到几个选项。选择最匹配你意图的那个。

**如果项目没有模板**（这是你需要这份指南的原因之一），继续看下一步。

### 第 4 步：写标题（Title）

标题要简短、清晰。让开发者一眼就知道你想说什么。

| 场景 | 好的标题 | 不好的标题 |
|:---|:---|:---|
| 报告 bug | `天气指令返回了去年的数据` | `坏了` |
| 提功能需求 | `能否增加"潮汐时刻"查询？` | `加个功能` |
| 贡献经验 | `[经验贡献] 盆栽绿萝叶片发黄急救` | `我有个想法` |
| 参与讨论 | `关于文贝分配的几点建议` | `我觉得不对` |

### 第 5 步：写正文（Description）

正文要包含足够的信息让开发者理解你。以下是每个场景的写作模板。

---

#### 📋 模板 1：报告 Bug

```markdown
## 我遇到的问题

（用一句话描述你遇到了什么问题）

例如：我使用"指令:基础应用;天气查询,明天,威海"，返回的数据是去年的。

## 我是怎么操作的

1. 我打开了（什么页面 / 发送了什么请求）
2. 我输入了（什么内容）
3. 我点击了（什么按钮）

## 我期望的结果

（描述你期望看到什么）

## 实际出现的结果

（描述实际上看到了什么。如果有截图，可以拖拽到这里）

## 我的环境

- 使用的设备：（电脑 / 手机）
- 使用的浏览器：（Chrome / Safari / 微信内置浏览器）
- 发生时间：（大概什么时间）
```

---

#### 📋 模板 2：提功能需求

```markdown
## 我想要什么功能

（用一句话描述你希望增加什么功能）

例如：希望增加一个"潮汐时刻"查询指令，输入日期和地点，返回涨潮和退潮时间。

## 这个功能为什么有用

（描述谁会用到它，解决什么问题）

例如：我住在一个渔村，很多渔民出海前需要知道潮汐时间。
目前他们只能用老经验估算，或者去专门的网站查，很不方便。

## 我已经尝试过的方法

（如果有的话）

## 我对这个功能的了解

（如果你有这个领域的经验，写出来。这是你的价值所在！）

例如：我父亲是渔民，家里有 30 年的潮汐观测笔记。
我可以提供潮汐计算的规则和经验数据。
```

---

#### 📋 模板 3：贡献经验

```markdown
## 我有什么经验可以分享

（用一句话介绍你的专长）

例如：我是花店老板，有 15 年盆栽养护经验。可以分享常见盆栽的急救方法。

## 这段经验能解决什么问题

（描述谁会受益）

例如：很多人养绿萝，叶片发黄了不知道怎么处理。
网上信息太杂，有的说过度浇水，有的说缺水，让人困惑。

## 经验大概包括哪些内容

- 叶片发黄的可能原因（3-5 种）
- 每种原因的识别方法
- 对应的处理方法
- 恢复期的注意事项

## 我需要怎么帮忙

- [ ] 我不知道怎么格式化——请 Agent 帮我转化为 Markdown
- [ ] 我可以口头描述，请帮我整理成文字
- [ ] 我已经写好了文字，请帮我审查

## 联系方式（可选）

如果开发者需要更多信息，可以通过以下方式联系我：
- Email：xxx@xxx.com
- 微信：xxx
```

---

#### 📋 模板 4：参与讨论

```markdown
## 我在讨论什么

（链接到你要讨论的 Issue 或 PR）

例如：关于 #55 中的文贝分配机制

## 我的观点

（用 2-3 句话总结你的看法）

## 我为什么这么认为

（基于你的经验、观察或逻辑）

## 我的建议

（如果有的话，提出具体的改进方案）

## 我的背景

（让其他人知道你从什么角度出发）

例如：我在传统制造业工作了 20 年，对劳动价值计量有自己的理解。
```

---

#### 📋 模板 5：申请成为服务商

```markdown
## 我想提供什么服务

（用一句话描述）

例如：我想注册一个"室内植物诊断"指令，帮助用户通过文字描述来判断植物问题。

## 服务的基本信息

- **指令领域**：（如 家庭园艺 / 医疗健康 / 教育培训）
- **指令动作**：（如 盆栽急救 / 方言翻译）
- **输入参数**：（用户需要提供什么信息）
- **输出结果**：（用户会得到什么）

## 我的资质

（为什么你有资格提供这个服务）

例如：我经营一家花店 15 年，服务过 5000+ 位顾客。
系统学习过植物病理学，有 3 本相关书籍的出版经验。

## 服务如何实现

- [ ] 我可以自己开发（有编程能力）
- [ ] 我需要 Agent 辅助开发（我提供经验，Agent 写代码）
- [ ] 我需要项目方协助（请指派开发者对接）

## 计费意愿

- [ ] 我想免费提供服务
- [ ] 我想按次收费
- [ ] 我想按月/年订阅
- [ ] 我还没想好
```

---

### 第 6 步：提交

写完标题和正文后，点击页面底部的绿色 **「Submit new issue」** 按钮。

✅ 完成！你刚刚在 GitHub 上创建了你的第一条 Issue。

### 第 7 步：等待回复

- 项目维护者（或 AI 协作者）会在 1-3 天内回复你的 Issue
- 你注册 GitHub 时用的邮箱会收到通知
- 你也可以在 GitHub 页面上的 **「Notifications」**（铃铛图标）里看到回复

## 五、在已有 Issue 下参与讨论（5 分钟）

你不需要每次都说新东西——在别人的讨论下留言也是一种重要的参与。

### 操作步骤

1. 点击进入你感兴趣的 Issue
2. 滚动到页面底部
3. 在 **「Write」**（写评论）框中输入你的想法
4. 点击绿色的 **「Comment」** 按钮

### 评论礼仪

| ✅ 应该做的 | ❌ 不应该做的 |
|:---|:---|
| 「我有类似的经历，补充一点……」 | 「+1」（只写这个没有任何信息量） |
| 「我不同意第 3 点，因为……」 | 「你们都不懂」（人身攻击） |
| 「我在另一个项目里见过类似的方案，链接是……」 | 重复别人已经说过的话 |
| 用 emoji 反应表达态度（👍❤️👀） | 偏离主题的闲聊 |

## 六、GitHub 常用术语（小词典）

| 英文术语 | 中文 | 用你能理解的话 |
|:---|:---|:---|
| **Repository** | 仓库 | 项目的「文件夹」，里面放着所有代码和文档 |
| **Issue** | 议题 | 留言簿上的留言——报告问题、提建议、分享经验 |
| **Pull Request (PR)** | 拉取请求 | 「我改好了，请审查并合并进去」——通常由会编码的人使用 |
| **Fork** | 分叉 | 把别人的项目复制一份到你自己名下 |
| **Star** | 星标 | 收藏/点赞一个项目 |
| **Watch** | 关注 | 订阅一个项目的更新通知 |
| **Commit** | 提交 | 保存一次修改 |
| **Branch** | 分支 | 项目的「草稿本」——在上面改东西不影响正式版本 |
| **Merge** | 合并 | 把草稿本上的修改并入正式版本 |
| **Maintainer** | 维护者 | 项目的管理员 |
| **Contributor** | 贡献者 | 参与过项目的人（包括提 Issue 的你！） |
| **Markdown** | — | 一种简单的文本格式，用 `#` 做标题、用 `-` 做列表 |

## 七、下一步：从参与者到贡献者

参与过一次 Issue 之后，你可以：

1. **持续关注**：点击项目的「Watch」按钮，收到更新通知
2. **点亮 Star**：点击右上角的 ⭐，表示你支持这个项目
3. **分享给更多人**：把项目链接发给和你一样有技能但不会编程的朋友
4. **深入参与**：当你熟悉 GitHub 后，可以学习创建 Pull Request

如果你想把经验变成可交易的工具，可以阅读 text-cli 的经验转化指南：[`Markdown2Text-cli_CN.md`](https://github.com/weihai-limh/text-cli/blob/main/docs/CN/Markdown2Text-cli_CN.md)

## 八、常见问题

**Q: 我写中文可以吗？**
A: 完全可以。text-cli 是中文优先的项目，直接用中文写 Issue 和评论。

**Q: 我害怕写错怎么办？**
A: GitHub 允许你随时编辑自己的 Issue 和评论。写错了改一下就好。而且很多项目（包括 text-cli）有 AI 协作者帮助理解和整理你的内容。

**Q: 我不知道用哪个模板怎么办？**
A: 选一个最接近的。如果都不匹配，选「提功能需求」或直接写一段话描述你的想法。开发者会帮你归类。

**Q: 我提交后没人理我？**
A: 大部分开源项目的维护者会在 1-3 天内回复。如果超过一周没有回复，可以在 Issue 下礼貌地留言提醒（如「请问有人看到这个吗？」）。

**Q: 我能在手机上操作吗？**
A: 可以。GitHub 有手机版网页。在手机浏览器中打开 github.com，操作步骤一样。

## 九、为你的项目添加 Issue 模板

如果你是项目维护者，想为非编码者提供更好的参与体验：

1. 在你的项目仓库中创建 `.github/ISSUE_TEMPLATE/` 目录
2. 从本仓库的 [`issue-templates/CN/`](./issue-templates/CN/) 目录复制模板文件
3. 修改模板中的项目名称和链接
4. 如需英文模板，参考 [`issue-templates/EN/`](./issue-templates/EN/)（待翻译）
