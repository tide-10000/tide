- 🧠 有独特的经验（花店老板、老渔民、维修师傅）但不知道怎么分享

### 你会学到

1. GitHub 是什么（用图书馆来比喻，不拽术语）
2. 创建 GitHub 账号（5 分钟，分步指引）
3. 创建你的第一条 Issue（10 分钟，5 种场景模板）
4. 如何在别人的讨论下留言
5. GitHub 常用术语小词典
6. 常见问题解答

### 五种参与场景

| 场景 | 你会得到什么 |
|:---|:---|
| 🐛 报告 Bug | Bug 报告写作模板——让开发者一眼就懂 |
| 💡 提功能需求 | 从「能不能加个 XX」到一份清晰的需求文档 |
| 🧠 贡献经验 | 把你的独门绝活变成开源项目的一部分 |
| 💬 参与讨论 | 在别人的提案下发表你的专业意见 |
| 🚀 成为服务商 | 把技能注册为指令，按次收费 |

### 这份指南的特点

- 🇨🇳 **中文优先**——用你看得懂的语言
- 🎯 **面向零基础**——不需要任何编程概念
- 🔓 **通用设计**——任何开源项目都可以链接到这里
- 🤖 **AI 友好**——每个模板都包含「Agent 辅助」标记，让 AI 协作者知道如何帮你
